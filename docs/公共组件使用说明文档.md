公共目录职责边界见 [公共能力目录说明.md](./公共能力目录说明.md)。新增业务页面前，应先确认本文和公共能力目录中是否已有可复用组件或工具。

## 1. HeaderTabs (顶部 Tab 标签组件)

用于页面顶部进行扁平块状的状态维度切换。激活态底色使用 Element Plus 预设变量。

- **文件路径**: `src/components/HeaderTabs/index.vue`
- **主题设计**:
  - 组件整体高度固定为 `38px`，外部圆角设为 `100px`，整体背景色为白色（`var(--el-bg-color)`）。
  - 高亮（Active）元素的圆角（`border-radius`）同步设置为胶囊形状的 `100px`。
  - 激活态的高亮底色动态使用全局变量（如成功色 `var(--el-color-success)` 或主色），非激活态字体颜色适配 `var(--el-text-color-regular)`。

### API 说明

#### Props

| 属性名       | 类型                                                               | 必填 | 默认值 | 说明                                             |
| :----------- | :----------------------------------------------------------------- | :--: | :----- | :----------------------------------------------- |
| `modelValue` | `string \| number`                                                 |  是  | -      | 当前选中的 Tab 值，支持 `v-model`                |
| `tabs`       | `Array<{ label: string; name: string \| number; count?: number }>` |  是  | -      | 标签选项数组，`count` 为该标签条数，默认值为 `0` |
| `showCount`  | `boolean`                                                          |  否  | `true` | 是否展示每种状态下的条数角标                     |

#### Events

| 事件名   | 参数                        | 说明                     |
| :------- | :-------------------------- | :----------------------- |
| `change` | `(value: string \| number)` | 切换 Tab 时触发 the 事件 |

### 使用用例

```vue
<template>
  <HeaderTabs v-model="activeStatus" :tabs="statusTabs" :show-count="true" @change="handleTabChange" />
</template>

<script setup lang="ts">
import { ref } from 'vue';
import HeaderTabs from '@/components/HeaderTabs/index.vue';

const activeStatus = ref('all');
const statusTabs = [
  { label: '全部', name: 'all', count: 32 },
  { label: '待处理', name: '待处理', count: 4 },
  { label: '进行中', name: '进行中', count: 16 },
  { label: '已完成', name: '已完成', count: 7 },
  { label: '已延期', name: '已延期', count: 3 }
];

const handleTabChange = (val: string | number) => {
  console.log('当前选中的Tab是:', val);
  // 执行查询过滤逻辑...
};
</script>
```

## 3. SearchHeader (配置化搜索栏组件)

通过传入配置项数组，在组件内统一根据字段类型自动渲染对应的过滤控件，并在搜索或重置时抛出表单数据。组件自带内部的响应式 `formData` 状态。

- **文件路径**: `src/components/SearchHeader/index.vue`
- **主题设计与特性**:
  - **自适应折叠布局**：采用 `ResizeObserver` 结合视口宽度变化实时自动计算可见项数并提供了防挂载时差保护，当表单项换行时自动展示“展开/折叠”按钮，对多搜索项支持极佳。
  - 支持 `theme` 设置主题样式：`'default' | 'card'`。
  - 支持 `showBorderBottom` 来控制底部分割线的显示。

### API 说明

#### Props

| 属性名             | 类型                         | 必填 | 默认值      | 说明                                     |
| :----------------- | :--------------------------- | :--: | :---------- | :--------------------------------------- |
| `formItems`        | `FormItem[]`                 |  是  | `[]`        | 过滤搜索项的配置数组，详见 FormItem 结构 |
| `showSearchBtn`    | `boolean`                    |  否  | `true`      | 是否显示查询与重置按钮                   |
| `showBackBtn`      | `boolean`                    |  否  | `false`     | 是否显示“返回”按钮，点击后自动回退上一页 |
| `labelWidth`       | `string`                     |  否  | `''`        | 标签宽度                                 |
| `labelPosition`    | `'left' \| 'right' \| 'top'` |  否  | `'right'`   | 标签对齐位置                             |
| `showBorderBottom` | `boolean`                    |  否  | `false`     | 是否显示底部分割边框                     |
| `theme`            | `'default' \| 'card'`        |  否  | `'default'` | 搜索栏风格样式主题                       |

##### FormItem 类型定义

```typescript
interface FormItem {
  key: string; // 该项在 formData 中对应的属性键值
  label: string; // 前置标签文案
  type: string; // 组件类型，支持: 'select' | 'input' | 'cascader' | 'treeSelect' | 'date-picker' | 'datetimerange' | 'daterange'
  placeholder?: string; // 输入框占位文案
  width?: string; // 控件宽度，例如 '160px'
  options?: Array<{ label: string; value: any; children?: any[] }>; // 选项数据源（适用于 select/cascader/treeSelect 等）
  elementAttrs?: Record<string, any>; // 穿透给 Element Plus 底层组件的额外属性（如 startPlaceholder/endPlaceholder 等）
  valueFormat?: string; // 日期格式化配置，例如 'YYYY-MM-DD'
  defaultValue?: any; // 默认初始值
  disabled?: boolean; // 是否禁用
}
```

#### Events

| 事件名           | 参数                              | 说明                                                         |
| :--------------- | :-------------------------------- | :----------------------------------------------------------- |
| `formDataChange` | `(formData: Record<string, any>)` | 点击搜索按钮或在输入框回车时触发，携带当前全部表单数据       |
| `formDataReset`  | `(formData: Record<string, any>)` | 点击重置按钮时触发，将内部字段置为 `null` 并抛出当前表单数据 |

### 使用用例

```vue
<template>
  <SearchHeader :form-items="searchFields" @form-data-change="onSearch" @form-data-reset="onReset" />
</template>

<script setup lang="ts">
import { ref, computed } from 'vue';
import SearchHeader from '@/components/SearchHeader/index.vue';

const queryParams = ref({
  leader: '',
  dateRange: null
});

// 配置式搜索项定义
const searchFields = computed(() => [
  {
    key: 'leader',
    label: '负责人',
    type: 'select',
    placeholder: '请选择负责人',
    width: '160px',
    options: [
      { label: '李云睿', value: '李云睿' },
      { label: '张华', value: '张华' }
    ]
  },
  {
    key: 'dateRange',
    label: '截止日期范围',
    type: 'daterange',
    width: '240px',
    valueFormat: 'YYYY-MM-DD',
    elementAttrs: {
      startPlaceholder: '请选择日期',
      endPlaceholder: '请选择日期'
    }
  }
]);

const onSearch = (data: any) => {
  queryParams.value.leader = data.leader || '';
  queryParams.value.dateRange = data.dateRange || null;
  console.log('执行搜索，当前检索参数:', queryParams.value);
};

const onReset = (data: any) => {
  queryParams.value = { leader: '', dateRange: null };
  console.log('搜索参数已重置');
};
</script>
```

## 4. CommonTable (通用表格组件)

参考 `coordinate` 原生精美样式封装的表格组件，强制约束行高为 `78px`，表头背景色 `#F4F5F7`。默认高度随内容自适应，在内容超出时自动占满父容器剩余空间并产生内部滚动条。

- **文件路径**: `src/components/CommonTable/index.vue`
- **主题设计**: 表头底色绑定 `var(--el-fill-color-light)`，表头及表格文字颜色取自 `var(--el-text-color-primary)`，行高写死于局部 CSS `:deep()` 中以强制覆盖 Element 原生高度。

### API 说明

#### Props

| 属性名      | 类型               | 必填 | 默认值   | 说明                                                                                                |
| :---------- | :----------------- | :--: | :------- | :-------------------------------------------------------------------------------------------------- |
| `data`      | `any[]`            |  是  | -        | 表格的数据源数组                                                                                    |
| `columns`   | `TableColumn[]`    |  是  | -        | 列配置参数数组，配置说明见下方                                                                      |
| `loading`   | `boolean`          |  否  | `false`  | 是否显示表格加载动画                                                                                |
| `rowKey`    | `string`           |  否  | `'id'`   | 每行数据的唯一键标识                                                                                |
| `height`    | `string \| number` |  否  | -        | 表格的固定高度，设置后高度被固定（内容少时也不会收缩）                                              |
| `maxHeight` | `string \| number` |  否  | `'100%'` | 表格的最大高度。默认为 `'100%'`，支持内容少时自适应收缩，内容多时最大占满父级剩余空间并触发内部滚动 |
| `showIndexColumn` | `boolean` | 否 | `false` | 是否显示公共序号列 |
| `indexLabel` | `string` | 否 | `'序号'` | 序号列表头 |
| `indexWidth` | `string \| number` | 否 | `70` | 序号列宽度 |
| `indexStart` | `number` | 否 | `0` | 序号起始偏移，分页列表传 `(currentPage - 1) * pageSize` |

##### TableColumn 类型定义

```typescript
interface TableColumn {
  prop?: string; // 数据源映射字段（Slot列可不填）
  label: string; // 列标题
  minWidth?: string | number; // 最小宽度
  width?: string | number; // 固定宽度
  slotName?: string; // 作用域插槽名称（如需要特殊渲染）
  showOverflowTooltip?: boolean; // 超出长度是否 tooltip 气泡提示
  fixed?: boolean | 'left' | 'right'; // 列固定配置
  className?: string; // 列单元格自定义类名
}
```

#### Slots

支持任意定义在 `columns` 中对应的 `slotName`。插槽接收数据行上下文 `{ row, $index, column }`。

### 使用用例

```vue
<template>
  <div style="height: 500px">
    <!-- 父容器须设定高度以配合 height="100%" 自适应 -->
    <CommonTable :data="tableData" :columns="tableColumns" :loading="loading" row-key="id">
      <!-- 自定义渲染状态列 -->
      <template #statusSlot="{ row }">
        <el-tag :type="row.status === '已完成' ? 'success' : 'warning'">
          {{ row.status }}
        </el-tag>
      </template>

      <!-- 自定义操作列 -->
      <template #actionsSlot="{ row }">
        <el-button link type="primary" @click="handleEdit(row)">编辑</el-button>
        <el-button link type="danger" @click="handleDelete(row)">删除</el-button>
      </template>
    </CommonTable>
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue';
import CommonTable from '@/components/CommonTable/index.vue';

const loading = ref(false);
const tableData = ref([
  { id: '1001', code: 'PRJ-001', name: '数据模型设计', status: '已完成' },
  { id: '1002', code: 'PRJ-002', name: '系统安全评测', status: '进行中' }
]);

const tableColumns = [
  { prop: 'code', label: '编号', minWidth: 120 },
  { prop: 'name', label: '任务名', minWidth: 200, showOverflowTooltip: true },
  { label: '状态', minWidth: 100, slotName: 'statusSlot' },
  { label: '操作', width: 150, fixed: 'right', slotName: 'actionsSlot' }
];

const handleEdit = (row: any) => {
  console.log('编辑:', row);
};

const handleDelete = (row: any) => {
  console.log('删除:', row);
};
</script>
```

## 5. CommonDivider (通用分割线组件)

用于在页面中绘制分割线。支持水平方向（可设置负的外边距来“吃掉”父容器的 padding 空间实现满宽贴边）与垂直方向（高度自适应拉满，适用于左右分栏隔离）。

- **文件路径**: `src/components/CommonDivider/index.vue`
- **设计**:
  - 水平模式：默认高度 `1px`，默认偏移量 `30px`，自适应带 padding 的面板进行贴边穿透。
  - 垂直模式：宽度固定 `1px`，高度自适应伸缩拉满（`align-self: stretch`），默认左右外边距 `20px`。

### API 说明

#### Props

| 属性名             | 类型               | 必填 | 默认值         | 说明                                                                                                                                 |
| :----------------- | :----------------- | :--: | :------------- | :----------------------------------------------------------------------------------------------------------------------------------- |
| `direction`        | `'horizontal' \| 'vertical'` |  否  | `'horizontal'` | 分割线方向。可选值为 `'horizontal'`（水平模式）或 `'vertical'`（垂直模式）                                                           |
| `offset`           | `number \| string` |  否  | `30`           | 水平模式下左右穿透负外边距的大小。传入数字自动补 `px`。应与父容器左右 `padding` 一致以实现完美穿透贴边。                             |
| `verticalMargin`   | `number \| string` |  否  | `20`           | 水平线时的上下垂直外边距。垂直线模式下若未设置 `verticalOffset` 时，亦作为上下的默认外边距。                                          |
| `horizontalMargin` | `number \| string` |  否  | `20`           | 垂直模式下分割线的左右水平外边距（数字默认 `px`）                                                                                    |
| `verticalOffset`   | `number \| string` |  否  | -              | 垂直模式下上下的穿透偏移量。传入数字自动补 `px`。用于通过负的 margin 穿透父容器的 padding 空间，实现垂直线顶格拉满。                 |
| `color`            | `string`           |  否  | -              | 分割线背景颜色，默认使用 `var(--el-border-color-lighter)`                                                                            |

### 使用用例

#### 1. 水平贴边分割线

```vue
<template>
  <div class="main-card-panel">
    <!-- 此处有 padding: 30px -->
    <div>上方内容</div>

    <!-- 穿透 30px padding 的满宽水平分割线 -->
    <CommonDivider :offset="30" :verticalMargin="16" />

    <div>下方内容</div>
  </div>
</template>

<script setup lang="ts">
import CommonDivider from '@/components/CommonDivider/index.vue';
</script>

<style scoped>
.main-card-panel {
  background: #ffffff;
  padding: 30px;
  border-radius: 12px;
}
</style>
```

#### 2. 垂直分栏分割线

```vue
<template>
  <div class="split-panel">
    <div class="left-box">左侧栏</div>

    <!-- 垂直分割线，左右间距为 24px，使用 verticalOffset 穿透上下 24px padding，达到完全顶格拉满 -->
    <CommonDivider direction="vertical" :verticalOffset="24" :horizontalMargin="24" />

    <div class="right-box">右侧栏</div>
  </div>
</template>

<script setup lang="ts">
import CommonDivider from '@/components/CommonDivider/index.vue';
</script>

<style scoped>
.split-panel {
  display: flex;
  background: #ffffff;
  padding: 24px;
}
.left-box {
  flex: 1;
}
.right-box {
  width: 300px;
}
</style>
```

## 6. Pagination (分页组件)

封装 Element Plus 分页组件，提供统一的样式交互、事件中继机制，以及默认的 `page-sizes` 数组配置。

- **文件路径**: `src/components/Pagination/index.vue`
- **主题设计**: 
  - 激活态页码高亮颜色动态使用 `var(--el-color-success)` 或 `var(--el-color-primary)`。
  - 支持 `'default'` 与 `'success'` 两种主题类型。
  - 当启用 `'success'` 时高亮背景色和文字颜色为一致的淡绿色。

### API 说明

#### Props

| 属性名           | 类型                   | 必填 | 默认值                               | 说明                                                                                          |
| :--------------- | :--------------------- | :--: | :----------------------------------- | :-------------------------------------------------------------------------------------------- |
| `currentPage`    | `number`               |  是  | -                                    | 当前激活页码，支持双向绑定                                                                    |
| `pageSize`       | `number`               |  是  | -                                    | 每页显示数据条数，支持双向绑定                                                                |
| `total`          | `number`               |  是  | -                                    | 总数据记录数                                                                                  |
| `pageSizes`      | `number[]`             |  否  | `[10, 20, 30, 50]`                   | 页面大小的可选下拉数组列表，默认约束为该数组                                                  |
| `layout`         | `string`               |  否  | `'sizes, prev, pager, next, jumper'` | 分页排版布局。默认已是不含 total 的紧凑组件，不传即为该默认值                                 |
| `paginationType` | `'default' \| 'success'` |  否  | `'default'`                          | 分页高亮主题类型。为 `'success'` 时激活态为淡绿色高亮以匹配全局样式，与 `'default'` 产生区别。 |

## 7. usePagination Hook (通用分页组合式函数)

为了避免在大量的列表页中重复书写声明分页状态与事件绑定等样板代码，特提炼出该 Hook 组合式函数以接管所有分页相关的底层流转。它在内部使用 `watch` 深度监测页码/页大小的修改，直接触发数据请求回调，极大精简了父组件模板。

- **文件路径**: `src/hooks/usePagination.ts`

### API 规格说明

#### 传入参数

- `fetchDataCallback` (`() => void | Promise<void>`): 查询获取列表数据的执行函数。在页码或每页大小变更时，该回调会被 Hook 的 `watch` 监听器自动调用。
- `defaultPageSize` (`number`): 默认的每页显示条数，默认为 `20`。

#### 返回结果

- `currentPage` (`Ref<number>`): 当前页码的响应式引用。
- `pageSize` (`Ref<number>`): 每页大小的响应式引用。
- `total` (`Ref<number>`): 总数据条数的响应式引用（在接口获取到 rows 后手动将 total 的值赋给该属性）。

### 综合使用范例 (Pagination + usePagination)

以下用例展示了如何结合使用组件与 Hook 以最大化简化业务代码：

````vue
<template>
  <div class="list-container">
    <!-- 数据列表展示 -->
    <CommonTable :data="tableData" :columns="columns" :loading="loading" />

    <!-- 分页配置：只需使用双向绑定更新 Ref 变量，优先采用 paginationType="success" 保持高亮颜色对齐 -->
    <Pagination
      v-model:current-page="currentPage"
      v-model:page-size="pageSize"
      :total="total"
      paginationType="success"
    />
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted } from 'vue';
import CommonTable from '@/components/CommonTable/index.vue';
import Pagination from '@/components/Pagination/index.vue';
import { usePagination } from '@/hooks/usePagination';

const loading = ref(false);
const tableData = ref([]);

// 定义列
const columns = [{ prop: 'title', label: '任务名' }];

// 1. 定义数据获取逻辑（供 Hook 自动监控调用）
const getList = async () => {
  loading.value = true;
  try {
    // 模拟请求接口，传入由 Hook 自动维护的页码参数
    const res = await apiGetTasks({
      pageNum: currentPage.value,
      pageSize: pageSize.value
    });
    tableData.value = res.rows;

    // 2. 将数据总量赋给 Hook 的 total
    total.value = res.total;
  } finally {
    loading.value = false;
  }
};

// 3. 使用 Hook 一键接管分页逻辑与状态，在页码或条数变化时会自动调用 getList
const { currentPage, pageSize, total } = usePagination(getList, 20); // 默认每页 20 条

onMounted(() => {
  getList();
});
</script>

## 9. 其他公共组件索引

以下组件在项目中已存在，生成新页面时应优先复用：

| 类型 | 组件 | 典型用途 |
| --- | --- | --- |
| 页面布局 | `FormPageLayout`、`DetailPageLayout` | 表单编辑页、详情页统一外壳 |
| 文件与媒体 | `FileUpload`、`ImageUpload`、`FilePreview`、`ImagePreview` | 上传、回显、预览附件和图片 |
| 业务选择 | `UserSelect`、`RoleSelect` | 人员和角色选择 |
| 数据回显 | `DictTag` | 字典标签回显 |
| 优先级卡片 | `PriorityCard` | 项目、重点任务等卡片列表 |
| 框架辅助 | `Breadcrumb`、`Hamburger`、`TopNav`、`Screenfull`、`SizeSelect`、`LangSelect`、`SvgIcon` | 布局和导航辅助 |
| 流程组件 | `Process/*` | 审批按钮、审批记录、流程图、提交校验等工作流场景 |

使用原则：

- 表单页先用 `FormPageLayout`，详情页先用 `DetailPageLayout`。
- 列表页先用 `SearchHeader`、`CommonTable`、`Pagination`，不要重复写搜索栏和分页。
- 上传、预览、人员选择、角色选择、字典标签、时间线必须优先使用现有组件。
- 只有公共组件 API 无法覆盖业务需求时，才在业务页面局部扩展；扩展具备复用价值后再沉淀为公共组件。

## 主题约束

后续项目会调整主题色风格，但页面布局风格保持不变。公共组件和业务页面应遵循：

- 主色只消费 `var(--el-color-primary)` 和 `src/utils/theme.ts` 派生出的变量。
- 不在组件中硬编码 `#409eff`、`#00b46e` 等主题主色。
- 不因为主题色变化改动组件结构、卡片层级、表格顺序、表单栅格和分页位置。
- 如需统一调整视觉细节，优先改公共组件或全局样式，不在多个业务页面重复覆盖。
