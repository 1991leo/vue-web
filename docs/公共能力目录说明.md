# 公共能力目录说明

更新时间：2026-06-26

## 目标

本文用于说明 `src/views/manage`、`src/api/manage`、`src/components`、`src/directive`、`src/enums`、`src/hooks`、`src/plugins`、`src/router`、`src/store`、`src/theme`、`src/utils` 的职责边界。新增业务页面、公共能力或主题调整前，先确认是否已有可复用实现。

## 目录边界

| 目录                   | 职责                                                       | 使用原则                                                                                                                |
| ---------------------- | ---------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| `src/views/manage`     | 后续新增业务页面目录                                       | 业务页面统一落在这里；旧 `systemCore/operations`、`systemCore/cockpit`、`systemCore/collaboration` 只作为迁移前风格参考 |
| `src/views/systemCore` | 平台基础页面                                               | 用户、角色、菜单、字典、租户、流程、系统监控、平台配置等基础能力保留在这里，不再新增业务页面                            |
| `src/api/manage`       | 后续新增业务接口                                           | 与 `src/views/manage` 模块对照，按 `{moduleName}/index.ts` + `types.ts` 组织                                            |
| `src/api/system`       | 平台基础接口                                               | 系统、权限、字典、租户、流程等基础能力接口保留在这里                                                                    |
| `src/components`       | 可复用 UI 组件、布局组件、选择组件、上传预览组件、流程组件 | 业务页面优先复用；只在多个页面出现稳定需求时新增公共组件                                                                |
| `src/directive`        | 全局自定义指令                                             | 权限、复制等 DOM 级通用能力放这里，业务判断不要写进指令                                                                 |
| `src/enums`            | 全局枚举                                                   | 语言、导航、菜单、响应码、侧边栏主题等跨模块枚举集中维护                                                                |
| `src/hooks`            | Vue 组合式函数                                             | 分页、loading、弹窗、CRUD 视图切换等可复用状态逻辑放这里                                                                |
| `src/plugins`          | 全局插件能力                                               | `$modal`、`$download`、`$tab`、`$auth`、缓存、svg icon 等全局挂载能力                                                   |
| `src/router`           | 路由初始化与动态路由                                       | 新增页面优先走路由/菜单配置，不在业务组件内硬编码导航结构                                                               |
| `src/store`            | Pinia 全局状态                                             | 用户、权限、字典、设置、标签页、通知等跨页面状态统一放 store                                                            |
| `src/theme`            | 项目主题 token                                             | 主色、语义色、中性色、圆角、阴影等系列色集中配置，运行时写入全局 CSS 变量                                               |
| `src/utils`            | 无 UI 的公共工具                                           | 请求、权限判断、若依工具、主题变量、字典、校验、加密、实时通信等                                                        |

## 业务页面与 API 目录

后续业务代码采用 `manage` 目录：

```text
src/views/manage/{category}/{featureName}/
├── index.vue
├── create.vue
├── detail.vue
├── config.ts
└── components/

src/api/manage/{moduleName}/
├── index.ts
└── types.ts
```

规则：

- `src/views/systemCore/operations`、`src/views/systemCore/cockpit`、`src/views/systemCore/collaboration` 后续会删除；新页面只参考其页面风格，不再写入这些目录。
- `src/views/systemCore` 与 `src/api/system` 只承载平台基础能力。
- 页面风格继续复用既有后台范式：标题、搜索、分割线、操作区、表格/卡片、分页；复杂表单和详情拆子组件。
- API 命名继续使用 `listXxx`、`getXxx`、`addXxx`、`updateXxx`、`delXxx`，列表取 `rows/total`，详情取 `data`。

## 公共组件

`src/components` 当前包含以下类型：

- 页面布局：`FormPageLayout`、`DetailPageLayout`、`PageTitle`、`CommonDivider`。
- 列表与筛选：`SearchHeader`、`CommonTable`、`Pagination`、`HeaderTabs`、`RightToolbar`。
- 文件与媒体：`FileUpload`、`ImageUpload`、`FilePreview`、`ImagePreview`。
- 业务选择与回显：`UserSelect`、`RoleSelect`、`DictTag`、`CommonTimeline`。
- 导航与框架：`Breadcrumb`、`Hamburger`、`TopNav`、`Screenfull`、`SizeSelect`、`LangSelect`、`SvgIcon`、`ParentView`、`iFrame`。
- 业务展示：`PriorityCard`、`Process/*` 流程相关组件。

新增页面时优先从上述组件组合，不要重新实现搜索栏、分页、详情布局、上传预览、用户/角色选择、字典回显和时间线。

## 指令与权限

`src/directive/index.ts` 负责统一注册指令。

- `src/directive/permission`：`v-hasPermi`、`v-hasRoles`，用于按钮和操作入口权限控制。
- `src/directive/common/copyText.ts`：复制文本能力。

新增、编辑、删除、导出等按钮必须接入 `v-hasPermi`。非按钮场景使用 `@/utils/permission` 的 `checkPermi` / `checkRole`。

## Hooks

`src/hooks` 当前常用能力：

- `usePagination(fetchFn, pageSize)`：统一维护 `currentPage`、`pageSize`、`total`，页码变化触发请求。
- `useLoading(asyncFn)`：托管异步任务 loading，避免重复提交。
- `useDialog`：统一弹窗开关与状态重置。
- `useCrudView`：管理列表、新建、编辑、详情等 CRUD 视图状态和路由参数清理。
- `useRate`：评分/比例类业务复用逻辑。

新增 hook 应保持无 UI、可测试、低耦合；涉及 DOM 或 Element Plus 实例时优先留在组件内。

## Store 与插件

`src/store/modules`：

- `user`：用户信息、角色、权限。
- `permission`：动态路由与菜单权限。
- `dict`：字典缓存。
- `settings`：主题、布局、语言、字号等系统设置。
- `app`：设备、侧边栏、语言和组件尺寸等应用状态。
- `tagsView`：页签状态。
- `notice`：通知类状态。

`src/plugins`：

- `modal`：统一消息、确认框、通知。
- `download`：文件下载。
- `tab`：页签和路由辅助。
- `auth`：权限辅助。
- `cache`：本地/会话缓存。
- `svgicon`：Element Plus 与本地 svg icon 注册。

业务代码优先使用已挂载插件和 store，不要重复封装全局弹窗、下载、权限和缓存。

## Utils

`src/utils` 中的重点文件：

- `request.ts`：axios 实例、token、业务 code 处理、错误提示。
- `permission.ts`：`checkPermi`、`checkRole`。
- `ruoyi.ts`：`parseTime`、`addDateRange`、`handleTree`、`tansParams` 等若依工具。
- `theme.ts`：根据 `src/theme` token 生成 Element Plus 与项目级 CSS 变量。
- `dict.ts`、`status.ts`、`taskCompany.ts`：字典、状态、业务映射工具。
- `auth.ts`、`crypto.ts`、`jsencrypt.ts`：鉴权与加密。
- `validate.ts`：通用校验。
- `websocket.ts`、`sse.ts`：实时通信。

新增纯函数或数据转换工具优先放 `src/utils`，复杂算法需要补 Vitest 单元测试。

## 主题调整边界

后续只修改主题色风格时，执行最小修改：

1. 默认色系、圆角、阴影优先改 `src/theme/index.ts`。
2. `src/settings.ts` 的 `theme` 只保留运行时主色入口，并默认引用 `src/theme` 的 `defaultThemeColor`。
3. 派生色继续由 `src/utils/theme.ts` 的 `handleThemeStyle` 生成。
4. 页面和公共组件只消费 `var(--el-color-primary)`、`var(--el-color-primary-light-*)`、`rgba(var(--el-color-primary-rgb), n)` 以及 `--app-*` 项目级变量。
5. 页面布局风格保持不变，不因为换主题色重写卡片结构、列表顺序、表单栅格、详情分栏、分页位置和公共组件 API。
6. 如确需调整公共视觉，只改 `src/theme`、公共组件或全局样式入口，避免在业务页面批量散落覆盖。

## 新增公共能力判断

满足以下条件再新增公共能力：

- 至少两个业务页面会复用，或一个核心流程中有稳定复用价值。
- API 足够通用，不绑定单一业务实体。
- 主题变量、权限、空态、loading、响应式和类型边界明确。
- 文档同步更新到本文件或 `docs/公共组件使用说明文档.md`。
