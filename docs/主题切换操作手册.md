# 主题切换操作手册

更新时间：2026-06-02

## 目标

本项目基于若依 Vue3 + Element Plus。主题切换的目标是：

- 主题主色只维护一个默认入口。
- Element Plus 组件统一消费 `--el-color-primary` 及其派生变量。
- 若依全局样式优先级高于 UI 组件库默认样式。
- 页面组件不写死 `#409EFF`、`#00B46E` 等主色。
- 后续只修改主题色风格时，页面布局风格保持不变，通过最小配置改动完成切换。

## 核心文件

| 文件                                      | 作用                                       |
| ----------------------------------------- | ------------------------------------------ |
| `src/settings.ts`                         | 默认主题色配置入口                         |
| `src/utils/theme.ts`                      | 根据主题色生成并写入 Element Plus CSS 变量 |
| `src/App.vue`                             | 应用启动时初始化主题变量                   |
| `src/main.ts`                             | 全局样式引入顺序控制                       |
| `src/assets/styles/index.scss`            | 项目全局样式入口                           |
| `src/assets/styles/element-ui.scss`       | 若依/项目对 Element Plus 组件的统一覆盖    |
| `src/assets/styles/variables.module.scss` | 若依布局、菜单、页面变量                   |
| `src/assets/styles/sidebar.scss`          | 侧边栏主题变量消费                         |
| `src/assets/styles/ruoyi.scss`            | 若依通用组件样式                           |

## 当前主题链路

1. 默认主题色在 `src/settings.ts` 中配置：

```ts
theme: '#00b46e';
```

2. 应用启动时，`src/App.vue` 调用：

```ts
handleThemeStyle(useSettingsStore().theme);
```

3. `src/utils/theme.ts` 写入以下 CSS 变量：

```ts
--el-color-primary
--el-color-primary-rgb
--el-color-primary-light-1 ... --el-color-primary-light-9
--el-color-primary-dark-1 ... --el-color-primary-dark-9
```

4. 全局样式和页面组件只使用 CSS 变量：

```scss
color: var(--el-color-primary);
background-color: rgba(var(--el-color-primary-rgb), 0.1);
```

## 样式引入顺序

`src/main.ts` 中的顺序必须保持如下：

```ts
import 'virtual:uno.css';
import 'element-plus/dist/index.css';
import 'element-plus/theme-chalk/dark/css-vars.css';
import '@/assets/styles/index.scss';
```

含义：

- `Element Plus` 默认样式先加载。
- 项目若依全局样式最后加载，保证最终覆盖权在项目侧。

不要再在 `src/assets/styles/index.scss` 中重复引入：

```scss
@use 'element-plus/dist/index.css';
```

## 禁止事项

### 1. 不要在全局样式中写死主色变量

不要这样写：

```scss
:root {
  --el-color-primary: #00b46e !important;
}
```

原因：这会和 `src/utils/theme.ts` 的运行时主题切换冲突。

### 2. 不要在业务页面写死主题色

不要这样写：

```scss
color: #409eff;
background: rgba(64, 158, 255, 0.1);
```

应该改为：

```scss
color: var(--el-color-primary);
background: rgba(var(--el-color-primary-rgb), 0.1);
```

## 修改默认主题色

只修改 `src/settings.ts`，这是默认主题色的最小修改入口：

```ts
theme: '#新的颜色';
```

示例：

```ts
theme: '#1678ff';
```

修改后重启开发服务或刷新页面，`App.vue` 会重新调用 `handleThemeStyle()` 初始化主题变量。

除非 `src/utils/theme.ts` 不能生成设计要求的派生色，否则不要改页面、公共组件结构或全局布局样式。主题换色不应影响 `PageTitle`、`SearchHeader`、`CommonTable`、`Pagination`、`FormPageLayout`、`DetailPageLayout` 等组件的布局顺序和 API。

## 最小修改策略

后续主题色调整按以下顺序处理：

1. 修改 `src/settings.ts` 的 `theme`。
2. 刷新页面，确认 `handleThemeStyle()` 已写入 `--el-color-primary`、`--el-color-primary-rgb` 和 light/dark 派生变量。
3. 若某个公共组件仍写死主色，只把该色值替换为主题变量。
4. 若多个页面有同类问题，优先改公共组件或 `src/assets/styles/*` 全局样式。
5. 不改页面布局风格，不批量重写业务页面结构。

## 运行时切换主题

设置面板中切换主题时，应调用：

```ts
settingsStore.theme = val;
handleThemeStyle(val);
```

如果新增主题切换入口，也必须走同一套逻辑，不要直接修改组件样式。

## 新增组件时的写法

### Element Plus 按钮

优先使用组件类型：

```vue
<el-button type="primary">提交</el-button>
<el-button type="primary" link>编辑</el-button>
```

不要手写按钮背景色。

### 自定义主题状态

```scss
.custom-primary-text {
  color: var(--el-color-primary);
}

.custom-primary-bg-light {
  background-color: rgba(var(--el-color-primary-rgb), 0.08);
}
```

### Hover 状态

```scss
.custom-link:hover {
  color: var(--el-color-primary-light-3);
}
```

## 排查主题不生效

### 1. 检查运行时 CSS 变量

在浏览器控制台执行：

```js
getComputedStyle(document.documentElement).getPropertyValue('--el-color-primary');
```

预期结果应为当前主题色。

继续检查：

```js
getComputedStyle(document.documentElement).getPropertyValue('--el-color-primary-rgb');
getComputedStyle(document.documentElement).getPropertyValue('--el-color-primary-light-3');
getComputedStyle(document.documentElement).getPropertyValue('--el-color-primary-dark-2');
```

### 2. 检查按钮最终变量

选中按钮 DOM，查看 `.el-button--primary` 上的变量：

```css
--el-button-bg-color
--el-button-border-color
--el-button-hover-bg-color
```

这些变量应引用或计算自 `--el-color-primary`。

### 3. 搜索硬编码主色

执行：

```bash
rg -n "#409eff|#409EFF|rgba\\(64, 158, 255|#00b46e|#00B46E|#00b96b" src
```

如果命中业务页面，应优先改为：

```scss
var(--el-color-primary)
rgba(var(--el-color-primary-rgb), 0.1)
```

### 4. 检查重复样式引入

执行：

```bash
rg -n "element-plus/dist/index.css" src vite.config.ts
```

预期：

- `element-plus/dist/index.css` 只应在 `src/main.ts` 出现一次。

## 提交前检查清单

- `npm run build:dev` 能通过。
- `src/main.ts` 样式顺序未被改乱。
- `src/settings.ts` 是默认主题色唯一配置入口。
- 没有新增 `--el-color-primary: xxx !important`。
- 没有新增 `#409eff`、`rgba(64, 158, 255, ...)` 作为主题色。
- 页面自定义样式使用 `var(--el-color-primary)`。
- 页面布局、公共组件 API、列表/表单/详情结构没有因换色被重写。

## 常见问题

### 按钮仍显示蓝色

优先检查：

1. `handleThemeStyle()` 是否执行。
2. `--el-color-primary` 是否还是 `#409eff`。
3. 是否有第三方 CSS 在项目全局样式之后加载。
4. 页面是否写了更高优先级的 `background` 或 `color`。

### 输入框 focus 颜色不对

检查 `.el-input__wrapper.is-focus` 的 `box-shadow` 是否来自：

```scss
var(--el-color-primary-light-8)
```

如果页面里用 `:deep(.el-input__wrapper)` 覆盖了 `box-shadow`，需要同步改为主题变量。
